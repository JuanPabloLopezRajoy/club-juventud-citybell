DOCUMENTACIÓN DEL SISTEMA

Sistema de Diseño

Documentación del sistema visual y de componentes desarrollado para el sitio institucional del Club Juventud de City Bell.

4.1 Sobre el caso

El proyecto consiste en el diseño y desarrollo del sitio institucional del Club Juventud de City Bell, una organización deportiva y social orientada a la formación de niños, niñas y adolescentes mediante la práctica del fútbol.

El sitio busca representar digitalmente la identidad del club, comunicar actividades, difundir noticias, facilitar el contacto con la comunidad e incentivar la incorporación de nuevos jugadores, socios y colaboradores.

El público destinatario está conformado principalmente por familias, jugadores actuales, futuros jugadores, vecinos de City Bell, socios y cualquier persona interesada en participar de las actividades deportivas de la institución.

Objetivos de la presencia web

  • Representar la identidad institucional.
  • Comunicar noticias y actividades.
  • Promover la inscripción de jugadores.
  • Favorecer el contacto con la comunidad.
  • Incrementar la cantidad de socios.
  • Difundir los valores deportivos y sociales del club.

ETAPA 1 · DECONSTRUCCIÓN

4.2 Análisis de referencia

El sistema fue diseñado a partir del análisis del sitio institucional del Club Argentino Juvenil de City Bell, identificando componentes reutilizables, patrones visuales y decisiones de diseño que sirvieron como base para la construcción del sistema propio.

Sistema analizado

Durante la etapa de deconstrucción se estudió la organización del sitio institucional del Club Argentino Juvenil de City Bell utilizando la metodología Atomic Design.

Se identificaron componentes reutilizables, relaciones jerárquicas entre ellos y la manera en que el sistema construía una identidad visual consistente mediante colores, tipografía, espaciados y composición.

Aprendizajes obtenidos

  • Uso consistente del color institucional.
  • Jerarquía tipográfica claramente definida.
  • Reutilización de tarjetas para distintos contenidos.
  • Navegación simple y accesible.
  • Organización modular de las secciones.
  • Uso repetitivo de bloques fácilmente escalables.

Decisiones aplicadas

  • Creación de variables CSS para todo el sistema.
  • Construcción mediante Atomic Design.
  • Uso de CSS Grid para la macroestructura.
  • Flexbox para componentes internos.
  • Responsive Mobile First.
  • Container Queries para las noticias.

Influencia sobre el sistema desarrollado

El análisis permitió definir una estructura modular basada en Atomic Design, construir un sistema de variables CSS para centralizar todas las decisiones visuales y desarrollar una interfaz completamente responsive utilizando HTML semántico, CSS Grid, Flexbox y Container Queries. Estas decisiones facilitaron la reutilización de componentes y garantizaron una identidad visual consistente en todas las páginas del proyecto.

IMPLEMENTACIÓN

4.5 Decisiones técnicas

Durante el desarrollo del sitio se tomaron distintas decisiones orientadas a construir un sistema escalable, reutilizable y fácil de mantener, respetando la metodología Atomic Design.

Grilla principal

La estructura general del sitio fue desarrollada con una estrategia Mobile First. Cada sección principal funciona como un organismo independiente y se organiza mediante CSS Grid y Flexbox según las necesidades de cada bloque.

Para los componentes internos se utilizaron grillas responsivas mediante display:grid, mientras que la distribución de elementos dentro de cada componente se resolvió con Flexbox cuando resultó más conveniente.

  • Mobile: diseño en una columna.
  • Tablet: redistribución de componentes.
  • Desktop: mayor cantidad de columnas y espacio disponible.

Container Queries

El componente .tarjeta-noticia utiliza Container Queries para modificar su disposición según el ancho de su propio contenedor y no únicamente del viewport.

Gracias a esta técnica la tarjeta puede mostrarse verticalmente en espacios reducidos y reorganizar imagen y contenido horizontalmente cuando dispone de mayor ancho, favoreciendo la reutilización del componente en distintos contextos.

Principal desafío

El mayor desafío consistió en transformar el análisis del sitio de referencia en un sistema propio, identificando componentes reutilizables, definiendo variables CSS y manteniendo coherencia visual entre todas las páginas.

También se buscó construir una arquitectura preparada para crecer, permitiendo incorporar nuevas noticias, galerías, actividades o campañas institucionales sin modificar la lógica general del sistema.
